# 🏥 检查系统的健康

> 瑞小美SCRM系统全面健康检查

---

## 🚀 执行策略：多 Agent 并行

启动 **4 个子 Agent** 并行检查（`readonly: true`，`model: fast`）：

| Agent | 检查范围 | 优先级 |
|-------|---------|-------|
| Agent 1 | 容器 + 数据库 + Nginx | P0 |
| Agent 2 | 企微回调 + 定时任务 | P1 |
| Agent 3 | AI服务 + SSL证书 | P1/P2 |
| Agent 4 | 资源监控 + 常见问题诊断 | P2 |

## 🔍 检查思路

### Agent 1：基础设施（P0）

**容器检查**：
- 所有容器是否运行中（无 Restarting/unhealthy）
- 各后端健康端点是否返回 200
- 热重载是否卡住（日志最后一行长时间停在 "Reloading..."）

**数据库检查**：
- MySQL 连接是否正常
- 连接数是否合理（< 100）

**Nginx检查**：
- 配置语法是否正确（nginx -t）
- 各子域名是否可访问
- 错误日志中是否有 "Connection refused"（DNS缓存问题）

### Agent 2：业务服务（P1）

**企微回调检查**：
- 各容器是否加载了 WECOM_*_TOKEN 和 WECOM_*_AES_KEY 环境变量
- 日志中是否有"签名验证失败"或"解密失败"

**定时任务检查**：
- 门户 /api/jobs 接口任务状态
- 是否有 status=running 超过 30 分钟的任务

### Agent 3：外部服务（P1/P2）

**AI服务检查**：
- 门户 /api/ai/internal/config 是否配置了 API Key
- 4sapi.com 和 openrouter.ai 是否可连接
- 最近 1 小时 AI 调用成功率

**SSL证书检查**：
- 各域名证书剩余天数（< 7天紧急，< 30天警告）
- Certbot 定时器是否运行

### Agent 4：资源与诊断（P2）

**资源监控**：
- 磁盘使用率（> 90%紧急，> 80%警告）
- 容器内存使用情况
- 日志目录大小

**常见问题诊断**（参考团队基线）：
- JWT_SECRET_KEY 各模块是否一致（应为 scrm-jwt-secret-key-2025）
- 容器间调用是否使用了错误的端口（应用内部端口，非主机映射端口）

## ⚡ 模式参数

- `--quick`：仅检查 P0（容器+数据库+Nginx）
- `--fix`：检查后尝试自动修复常见问题（重启卡住容器、reload nginx）

# 更新系统文档

> 根据本次操作内容，更新对应的系统文档。

## 文档目录

项目知识库路径：`/root/瑞小美AiOS/文档/`

| 分类 | 路径 | 包含文档 |
|-----|------|---------|
| 核心信念 | `文档/核心信念/` | 团队基线、技术栈标准、OAuth规范、AI接入规范、流式输出规范、企微托管技能规范 |
| 参考资料 | `文档/参考资料/` | 数据库配置、网络与域名、联调指南、Api文档、第三方对接 |
| 设计文档 | `文档/设计文档/` | 各功能设计方案（需维护验证状态头部） |
| 凭据与配置 | `文档/凭据与配置/` | 环境变量说明、开发环境凭据总览 |
| 变更日志 | `文档/变更日志/` | AI功能开发记录、各模块变更记录 |
| 执行计划 | `文档/执行计划/` | 进行中/已完成/技术债务 |
| 产品规格 | `文档/产品规格/` | 业务规则、总进度汇报 |
| 质量评分 | `文档/质量评分/` | 模块质量评分 |
| 项目鸟瞰 | 根目录 | `架构.md`（部署拓扑、模块依赖） |
| AI 入口 | 根目录 | `AGENTS.md`（知识库索引+编码规范，Cursor 原生读取） |

## 执行策略

1. 分析本次操作涉及哪些文档分类
2. 若涉及多个分类，使用 Task 工具并行启动子 agent 分别处理
3. 若变更了容器/端口/域名，同步更新 `架构.md`
4. 若新增/删除了文档文件，同步更新 `AGENTS.md` 知识库索引
5. 汇总更新结果

## 更新原则

- 按需更新，不扩散
- 追加优先，避免重复
- 重要更新标注日期
- 表格/列表优于长文
- 设计文档必须维护验证状态头部
- 文档不含示例代码，充分信任大模型

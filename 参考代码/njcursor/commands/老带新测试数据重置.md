清空老带新模块的邀请、奖励等业务流转数据，为新一轮测试做准备。

## 规则

1. **只清业务数据，保留配置和基础数据**：
   - 清空：invitations、rewards、coupons、arrivals、consumptions、consultant_assignments、notifications
   - 清空：sync_progress（历史同步记录）
   - 重置：channel_sync_cursor（同步游标恢复idle）
   - 重置：customers 表中测试产生的 `ldx_miniprogram_openid` 绑定
   - 重置：consultants 表的分配计数（today/total_assignments、arrivals、conversions）
   - 保留：customers（24万+客户数据）、consultants（顾问配置）、policies（政策配置）、welcome_messages（欢迎语模板）

2. **注意外键约束**：先关闭 `FOREIGN_KEY_CHECKS`，子表先清，主表后清，最后恢复。

3. **使用 TRUNCATE**：自增 ID 会一并重置，确保测试从干净的状态开始。

4. **清空后验证**：执行后需查询确认所有业务表为 0，配置表数据完好。

5. **数据库连接信息**：
   - 容器：`scrm_mysql`
   - 数据库：`scrm_content`
   - 用户：`scrm_user` / `ScrmUser2024Pass`

## 执行命令

```bash
docker exec scrm_mysql mysql -uscrm_user -pScrmUser2024Pass scrm_content -e "
SET FOREIGN_KEY_CHECKS = 0;

-- ========== 清空业务流转数据 ==========
TRUNCATE TABLE referral_rewards;
TRUNCATE TABLE referral_coupons;
TRUNCATE TABLE referral_arrivals;
TRUNCATE TABLE referral_consumptions;
TRUNCATE TABLE referral_consultant_assignments;
TRUNCATE TABLE referral_notifications;
TRUNCATE TABLE referral_invitations;

-- ========== 清空同步历史记录 ==========
TRUNCATE TABLE referral_sync_progress;

-- ========== 重置同步游标为初始状态 ==========
UPDATE referral_channel_sync_cursor SET last_id = 0, cycle = 0, status = 'idle', error_count = 0, last_error = NULL, last_run_at = NULL WHERE id = 1;

-- ========== 清除测试产生的小程序openid绑定 ==========
UPDATE referral_customers SET ldx_miniprogram_openid = NULL WHERE ldx_miniprogram_openid IS NOT NULL;

-- ========== 重置顾问分配计数 ==========
UPDATE referral_consultants SET today_assignments = 0, total_assignments = 0, total_arrivals = 0, total_conversions = 0, last_assignment_time = NULL;

SET FOREIGN_KEY_CHECKS = 1;

-- ========== 验证：业务表应为 0 ==========
SELECT '=== 应为 0 ===' AS section, '' AS c
UNION ALL SELECT 'referral_invitations', COUNT(*) FROM referral_invitations
UNION ALL SELECT 'referral_rewards', COUNT(*) FROM referral_rewards
UNION ALL SELECT 'referral_coupons', COUNT(*) FROM referral_coupons
UNION ALL SELECT 'referral_arrivals', COUNT(*) FROM referral_arrivals
UNION ALL SELECT 'referral_consumptions', COUNT(*) FROM referral_consumptions
UNION ALL SELECT 'referral_consultant_assignments', COUNT(*) FROM referral_consultant_assignments
UNION ALL SELECT 'referral_notifications', COUNT(*) FROM referral_notifications
UNION ALL SELECT 'referral_sync_progress', COUNT(*) FROM referral_sync_progress
UNION ALL SELECT 'ldx_openid绑定残留', SUM(ldx_miniprogram_openid IS NOT NULL) FROM referral_customers
UNION ALL SELECT '=== 应保留 ===' AS section, '' AS c
UNION ALL SELECT 'referral_customers', COUNT(*) FROM referral_customers
UNION ALL SELECT 'referral_consultants', COUNT(*) FROM referral_consultants
UNION ALL SELECT 'referral_policies', COUNT(*) FROM referral_policies
UNION ALL SELECT 'referral_welcome_messages', COUNT(*) FROM referral_welcome_messages
UNION ALL SELECT 'referral_channel_sync_cursor', COUNT(*) FROM referral_channel_sync_cursor;
"
```

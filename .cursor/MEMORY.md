## 经验教训

### 2026-03-20 Bun WebSocket 在代理环境下不可靠
- Bun 替换了 `ws` 库的实现，其 WebSocket 客户端在有 Mihomo/Clash 代理的环境下可能无法正常连接外部 WebSocket 服务
- **对策**：涉及 WebSocket 外部连接的功能，如果 Bun 下不工作，考虑用 Node.js 子进程执行

### 2026-03-20 Git HTTPS 经 Clash 代理时 SSL 握手失败
- 服务器上 git 走 Clash HTTP 代理（127.0.0.1:7897）时，OpenSSL 在 CONNECT 隧道内握手出现 `SSL_ERROR_SYSCALL`
- 直连 GitHub HTTPS 反而正常
- **解决**：在 `.git/config` 中为 GitHub 单独禁用 HTTP 代理 `[http "https://github.com/"] proxy =`

### 2026-03-20 语音文件删除时机
- 旧代码在语音识别失败后也删除音频文件，导致 Agent 拿到的路径指向已删除的文件
- **正确做法**：只在语音识别成功后才删除音频文件；失败时保留，让 Agent 可以回退处理

### 教训
- **2026-03-24**：**纯内存状态在重启后丢失** — `lastActiveChatId` 是内存变量，服务重启后所有定时任务飞书投递全部失败。关键运行时状态必须持久化到磁盘。
- **2026-03-24**：**文件监听 + 长任务 = 竞态** — 调度器 tick() 中某个任务完成后 save() 写 cron-jobs.json，触发文件监听 reload → reschedule → 新 tick()，而其他长任务仍在执行中，导致重复启动。必须有任务级和 tick 级的执行锁。
- **2026-03-24**：**`LD_PRELOAD` 是隐形毒药** — proxychains 通过 `LD_PRELOAD=libproxychains4.so` 劫持所有网络调用，子进程（Chromium、Python）都继承。清除 `HTTP_PROXY` 不够，必须同时清除 `LD_PRELOAD`。SLS 查询 502、全站巡检 Token 注入失败、企微白屏都是同一根因。
- **2026-03-24**：**"Connection stalled" 是代理不稳定，不是 Cursor 并行限制** — Cursor API 如果有并发限制会返回 429，不会表现为 stalled。不要在没有证据的情况下给出归因。
- **2026-03-24**：**机场订阅 UA 版本影响返回节点数** — `clash-verge/v2.2.0` 被机场认为太旧只返回 7 个节点，改为 `v2.3.0` 后返回 56 个（含台湾/日本/美国/德国）。
- **2026-03-24**：**SQLAlchemy Computed 列必须标记** — `chat_messages.to_userid` 是 MySQL STORED 生成列，但模型定义为普通 Column，INSERT 报错，每小时约 110 条消息同步失败。涉及三个模块。

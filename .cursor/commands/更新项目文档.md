# 更新系统文档

> 根据本次操作内容，更新对应的系统文档。

## 文档目录

自动扫描项目中的文档目录（常见位置：`docs/`、`文档/`、项目根目录的 `*.md`）。

常见的文档分类：

| 分类 | 说明 |
|-----|------|
| 核心规范 | 技术栈标准、编码规范、API 规范、架构约束 |
| 参考资料 | 数据库配置、网络与部署、联调指南、第三方对接 |
| 设计文档 | 仍在指导开发的架构设计 |
| 产品规格 | 业务规则、判断逻辑、决策标准 |
| 执行计划 | 进行中 / 技术债务 |
| 项目鸟瞰 | 架构图、部署拓扑、模块依赖关系 |
| 知识库索引 | `AGENTS.md`（AI 工具的入口索引） |

## 执行策略

1. 分析本次操作涉及哪些文档分类
2. 若涉及多个分类，使用 Task 工具并行启动子 agent 分别处理
3. 若变更了容器 / 端口 / 域名 / 部署配置，同步更新架构文档
4. 若新增 / 删除了文档文件，同步更新知识库索引
5. 口头汇总更新结果

## 更新原则

- 按需更新，不扩散
- 文档只反映当前状态，不写变更历史
- 表格 / 列表优于长文
- 文档不含代码示例，充分信任大模型
- **禁止生成**：变更日志、巡检报告、审计报告、测试报告、完成报告

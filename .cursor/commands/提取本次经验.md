# 经验提取

从本次会话中提取可复用的模式和经验，沉淀为团队知识。

## 与"规则沉淀"的区别

- 规则沉淀：解决具体问题后的复盘，产出"避坑规则"
- 经验提取：任何会话结束时的总结，产出"最佳实践"

## 执行步骤

1. 回顾本次会话的所有操作
2. 识别可复用的模式：
   - 解决某类问题的通用方法
   - 发现的高效工作流
   - 值得在其他项目中复用的做法
3. 判断经验级别：
   - 项目级 → 写入项目文档
   - 团队级 → 写入团队共享知识库
4. 用简洁的格式记录，标注触发场景和具体做法
5. 检查是否已有类似经验，避免重复

## 输出格式

**经验标题**
- 触发场景：什么时候用
- 具体做法：怎么做
- 效果：解决了什么问题

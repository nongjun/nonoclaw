# 从 Git 提取团队习惯

分析当前项目的 Git 提交历史，自动提取团队的编码习惯和工作模式，生成团队习惯文件。

## 触发场景

- 新项目接入 harness 时，快速了解团队已有的习惯
- 定期回顾，看看团队形成了哪些新的模式
- 新成员加入时，让 AI 自动学会团队的做事方式

## 分析内容

从 Git 历史中提取以下模式：

| 分析项 | 检测方法 |
|--------|----------|
| 提交规范 | 分析提交消息格式（是否用 feat:/fix: 等前缀） |
| 文件联动 | 哪些文件总是一起修改（改 A 必改 B） |
| 工作流序列 | 重复出现的文件变更模式（如：先建组件→写测试→导出） |
| 目录结构 | 项目的组织方式和命名约定 |
| 测试习惯 | 测试文件的位置、命名、覆盖情况 |
| 活跃模块 | 哪些目录修改最频繁 |

## 执行步骤

1. 读取最近 200 次 Git 提交（可用 `--commits 数量` 调整）
2. 分析提交消息、变更文件、时间分布
3. 识别重复出现的模式
4. 生成团队习惯文件，保存到：`文档/团队习惯.md`
5. 输出发现摘要

## 生成文件格式

```markdown
# 团队习惯

> 由 AI 从 Git 历史自动提取，最后更新：YYYY-MM-DD
> 分析了 N 次提交，时间范围：起始日期 ~ 结束日期

## 提交规范
{检测到的提交消息模式}

## 代码组织
{检测到的目录结构和命名约定}

## 工作流模式
{检测到的重复文件变更模式}

## 测试习惯
{检测到的测试约定}

## 活跃模块
{按修改频率排序的模块列表}
```

## 参数

- `--commits 数量`：分析的提交数（默认 200）
- `--since 日期`：只分析某日期之后的提交
- `--update`：更新已有的团队习惯文件（追加新发现，不覆盖）

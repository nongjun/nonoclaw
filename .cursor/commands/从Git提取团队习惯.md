# 从 Git 提取团队习惯

分析当前项目的 Git 提交历史，提取团队的编码习惯和工作模式，将可复用的模式写入 Cursor rules。

## 触发场景

- 新项目接入 harness 时，快速了解团队已有的习惯
- 定期回顾，看看团队形成了哪些新的模式

## 分析内容

从 Git 历史中提取以下模式：

| 分析项 | 检测方法 |
|--------|----------|
| 提交规范 | 分析提交消息格式（是否用 feat:/fix: 等前缀） |
| 文件联动 | 哪些文件总是一起修改（改 A 必改 B） |
| 目录结构 | 项目的组织方式和命名约定 |
| 测试习惯 | 测试文件的位置、命名、覆盖情况 |
| 活跃模块 | 哪些目录修改最频繁 |

## 执行步骤

1. 读取最近 200 次 Git 提交（可用 `--commits 数量` 调整）
2. 分析提交消息、变更文件、时间分布
3. 识别重复出现的模式
4. 将发现的模式**写入或更新** `.cursor/rules/` 中对应的规则文件（如提交规范写入基础规则，编码模式写入编码风格规则）
5. 在对话中输出发现摘要

**注意**：不生成独立的 `团队习惯.md` 文件。提取结果直接融入 Cursor rules，让 AI 自动遵循。

## 参数

- `--commits 数量`：分析的提交数（默认 200）
- `--since 日期`：只分析某日期之后的提交

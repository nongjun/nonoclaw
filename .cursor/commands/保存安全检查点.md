# 检查点

保存当前"验证通过"的状态，作为安全回退基线。

## 触发场景

- 大改动前，先存一个"安全点"
- 一个功能完成且验证通过后，标记里程碑
- 尝试风险较大的方案前

## 执行步骤

1. 先运行 /提交前全量检查 命令，确认当前状态全部通过
2. 如果验证不通过，提示用户先修复再存检查点
3. 验证通过后，记录以下信息：
   - 时间戳
   - Git commit hash（如果有）
   - 验证结果摘要
   - 当前正在做什么的简要描述
4. 将检查点信息保存到项目文档中

## 回退对比

存了检查点后，后续可以用来：
- 对比当前状态和检查点的差异
- 快速了解"从上个安全点到现在改了什么"
- 出问题时知道"回到哪个状态是安全的"

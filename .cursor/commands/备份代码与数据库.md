# 备份代码与数据库

> 将服务器上全部代码项目和 MySQL 数据库完整备份到数据盘，压缩存储

## 触发场景

- 大改动前保存完整快照
- 定期备份（每周 / 每月）
- 服务器迁移前
- 数据库结构变更前

## 备份位置

- **统一目录**：`/data/backups/YYYY-MM-DD/`
- 子目录：`db/`（数据库）、`code/`（代码）
- 每次备份按日期隔离，不覆盖历史备份

## 执行策略：多 Agent 并行

启动 **3 个子 Agent** 并行执行：

| Agent | 职责 | 预计耗时 |
|-------|------|---------|
| Agent 1 | MySQL 数据库导出 + gzip 压缩 | 2~3 分钟 |
| Agent 2 | 大型项目打包压缩（>10MB 的项目） | 1~2 分钟 |
| Agent 3 | 小型项目打包 + Git 项目提交推送 | <1 分钟 |

## 操作步骤

### 0. 环境探测（自动执行）

不硬编码任何路径或密码，全部自动发现：

- `df -hT` — 确认数据盘挂载点和剩余空间
- `source /root/瑞小美AiOS/.env` — 读取 RDS 数据库连接信息（DB_HOST/DB_USER/DB_PASSWORD/DB_NAME）
- `mysql -h"${DB_HOST}" -u"${DB_USER}" -p"${DB_PASSWORD}" -e "SELECT 1"` — 验证 RDS 连接可用
- 扫描 `/root/` 下的项目目录，按是否有 `.git` 分类
- 估算备份总大小，与数据盘剩余空间对比，空间不足则中止并提醒

### 1. 创建备份目录

```
/data/backups/YYYY-MM-DD/
├── db/          # 数据库备份
├── code/        # 代码压缩包
└── 备份清单.md   # 备份记录
```

### 2. 数据库备份

- 使用宿主机 `mysqldump` 直接连接阿里云 RDS 执行导出
- 参数：`--single-transaction`（InnoDB 热备不锁表）、`--routines`、`--triggers`、`--events`
- 通过 `gzip -9` 最大压缩
- 文件名格式：`{库名}_{HHMMSS}.sql.gz`

### 3. 代码项目打包

- 使用 `tar czf` 打包每个项目目录
- 排除无用目录：`node_modules`、`.next`、`__pycache__`、`*.pyc`、`.git`、`dist`、`.venv`、`venv`
- 每个项目生成独立的 `.tar.gz` 文件

### 4. Git 项目处理

- 检查有 `.git` 的项目是否有未提交文件
- 如果有未提交文件，执行 `git add -A` → `git commit` → `git push`
- 确保远程仓库有最新代码

### 5. 完整性验证

- 对所有 `.sql.gz` 文件执行 `gzip -t` 验证
- 对所有 `.tar.gz` 文件执行 `tar tzf` 验证
- 任何文件验证失败立即报警，重新备份该文件

### 6. 生成备份清单

在备份目录生成 `备份清单.md`，包含：
- 备份时间、服务器信息
- 每个备份文件的名称、原始大小、压缩后大小
- 数据库恢复命令
- 代码恢复命令

## 输出格式

```markdown
## 备份完成汇总

**备份位置**：/data/backups/YYYY-MM-DD/
**总大小**：X.X GB | **数据盘剩余**：XX GB

### 数据库

| 文件 | 原始大小 | 压缩后 |
|------|---------|--------|
| db/aios_XXXXXX.sql.gz | X.X GB | XXX MB |

### 代码（N 个项目）

| 文件 | 压缩后 |
|------|--------|
| code/项目名.tar.gz | XX MB |

### 验证结果

- N 个文件全部通过完整性校验 ✅
- Git 项目已推送至远程 ✅
```

## 磁盘空间不足时的处理

1. 列出历史备份及其大小：`du -sh /data/backups/*/`
2. 提示用户是否删除最旧的备份以腾出空间
3. 用户确认后才执行删除

## 恢复指南（备份清单中自动包含）

### 恢复数据库

```
source /root/瑞小美AiOS/.env && gunzip < /data/backups/日期/db/aios_XXXXXX.sql.gz | mysql -h"${DB_HOST}" -u"${DB_USER}" -p"${DB_PASSWORD}" "${DB_NAME}"
```

### 恢复代码

```
tar xzf /data/backups/日期/code/项目名.tar.gz -C /root/
```

## 注意事项

- 数据库连接信息从 `.env` 文件读取，不硬编码
- 数据库已迁移至阿里云 RDS，同时享受 RDS 自动备份保护
- 备份过程不停服，使用 `--single-transaction` 保证一致性
- 大项目和数据库并行备份，缩短总耗时
- 每次备份独立目录，不会覆盖之前的备份

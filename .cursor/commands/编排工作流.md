# 编排

将多个命令或 Agent 串联成一个完整工作流，顺序执行。

## 触发场景

- 需要按固定顺序执行一系列操作
- 重复性的工作流想一键完成

## 使用方式

告诉我你想串联的步骤，我会按顺序执行。例如：
- "先规划，再写代码，最后审查和测试"
- "先检查系统健康，再更新文档，最后 git 提交"

## 内置工作流模板

### 功能开发流
规划实施方案 → 实现 → 代码审查专家 → 测试后端Api → 提交前全量检查 → Git提交和推送

### 问题修复流
调试 → 修复 → 审查刚才的文件与操作 → 提交前全量检查 → 复盘并沉淀规则 → Git提交和推送

### 发布前检查流
提交前全量检查 → 基础设施健康检查 → 项目总监审视项目 → 更新项目文档 → Git提交和推送

## 执行原则

- 每步完成后展示结果，等确认再继续下一步
- 某步失败时暂停，询问是否跳过还是修复后继续
- 可以随时中断工作流

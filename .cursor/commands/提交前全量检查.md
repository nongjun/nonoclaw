# 提交前全量检查

提交代码前一键跑完所有检查，确认改动没有破坏任何东西。

## 触发场景

- 做完一波修改后，想确认一切正常
- 准备提交代码前的最后确认
- 合并分支前的完整性检查

## 检查项（按顺序执行）

1. **构建** — 项目能否正常构建/编译
2. **测试** — 所有测试是否通过
3. **类型检查** — 有无类型错误（TypeScript/Python 类型提示）
4. **代码规范** — Lint 检查是否通过
5. **安全扫描** — 调用安全卫士 Agent 检查密钥泄露和常见漏洞

## 执行策略

- 使用子 agent 并行执行互不依赖的检查项（如 Lint 和安全扫描可并行）
- 构建必须先通过，后续检查才有意义
- 每项标注 ✅ 通过 / ❌ 失败 / ⚠️ 警告

## 输出格式

```
| 检查项 | 状态 | 详情 |
|--------|------|------|
| 构建 | ✅/❌ | ... |
| 测试 | ✅/❌ | N 通过 / M 失败 |
| 类型检查 | ✅/❌ | ... |
| 代码规范 | ✅/⚠️ | ... |
| 安全扫描 | ✅/⚠️ | ... |

总结：X/5 通过
```

## 失败时

- 列出所有失败项的具体错误
- 按严重程度排序（构建 > 测试 > 类型 > 规范 > 安全）
- 询问用户是否要自动修复（调用构建修复师/调试专家）

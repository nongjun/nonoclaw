# 微信小程序开发模式

## 适用场景

开发微信小程序、使用 Uni-app 进行跨平台小程序开发时。

## 核心原则

- 体积敏感——小程序有 2MB 主包限制，分包后每个分包 2MB
- 性能优先——setData 是性能瓶颈，减少数据量和频率
- 审核友好——避免使用可能被拒绝的敏感 API 和内容
- 用户体验——加载快、交互流畅、符合微信设计规范

## 分包策略

- 主包只放首页和公共资源，控制在 2MB 以内
- 按功能模块分包，用户进入时才下载
- 公共组件和工具放在主包，业务页面放在分包

## 数据通信

- 页面间传参优先用 URL 参数，复杂数据用全局状态
- 与后端通信统一封装 request，处理 Token 刷新和错误提示
- 缓存策略：频繁读取的用 wx.getStorageSync，实时数据走接口

## 常见坑点

- 小程序不支持 DOM 操作，用数据驱动视图
- 部分 CSS 特性不支持（如 calc 在某些场景不生效）
- 支付、分享等需要在后台配置对应权限
- 图片必须使用 HTTPS 或 base64
- unionId 需要用户关注公众号或授权后才能获取

## Uni-app 注意事项

- 条件编译（#ifdef MP-WEIXIN）处理平台差异
- 生命周期和 Vue 生命周期的对应关系要清楚
- 原生小程序组件用 uni 前缀替代 wx 前缀

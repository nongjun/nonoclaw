# 文档园丁

知识库建立后的**唯一维护入口**。核心职责：让文档保持精简、准确、与代码一致。

## 核心原则

**绝对不生成任何临时性文档**（变更日志、巡检报告、审计报告、测试报告、完成报告）。发现问题直接修复，修复结果在对话中口头汇报即可。

可复用的经验和规则写入 Cursor rules（`.cursor/rules/`），不写独立 .md 文件。

## 子 agent 使用规范

所有涉及多文件、多模块的操作，必须使用 Task 工具启动子 agent 并行处理（每批最多 4 个）。

## 能力A：巡检

**触发**："文档巡检" / "跑一下文档园丁" / "检查文档"

### 第一步：运行扫描脚本

```bash
bash ~/.cursor/skills/文档园丁/scripts/scan.sh <项目根目录>
```

### 第二步：子 agent 并行修复

根据扫描结果，启动并行子 agent 分头直接修复：

| 问题类型 | 处置 |
|---------|------|
| 容器/端口/模块不一致 | 直接更新 `架构.md` 和 `AGENTS.md` |
| 凭据不一致 | 直接更新凭据文档 |
| 代码变更未同步文档 | 逐模块分析 git diff，直接更新对应文档 |
| 过期/空目录 | 直接删除 |
| AGENTS.md 索引与文档目录不同步 | 直接修正索引 |

### 第三步：口头汇报

在对话中汇总修复了什么，**不生成任何报告文件**。

---

## 能力B：整理散乱文档

**触发**："整理文档" / "整理XX模块的文档"

### 流程

1. 扫描目标范围内所有 `.md` 文件
2. 逐份判断：迁移 / 合并 / 删除
3. 并行子 agent 执行处置
4. 清理空目录，更新 `AGENTS.md` 索引
5. 口头汇报结果

### 处置标准

| 判断 | 动作 |
|------|------|
| 仍在指导开发的架构/规范 | 迁移到 `文档/` 对应子目录 |
| 人类审核过的业务决策/判断规则 | 保留并迁移 |
| 多份文档说同一件事 | 合并为一份 |
| 已落地的设计方案 | 删除（代码即文档） |
| 一次性分析/讨论/报告 | 删除 |
| 变更日志/巡检报告/审计报告 | 删除 |

---

## 能力C：审计模块旧文档

**触发**："审计XX模块的文档" / "梳理XX模块"

### 流程

1. 定位模块内所有 `.md` 文件
2. 子 agent 并行验证每份文档与代码的一致性
3. 识别人类决策内容（必须保留）vs 实现细节（代码已包含，可删除）
4. 直接执行处置，口头汇报结果

---

## 自动修复权限

| 操作 | 权限 |
|------|------|
| 更新架构.md / AGENTS.md | 直接修复 |
| 删除过期文档/空目录 | 直接修复 |
| 凭据/配置文档更新 | 修复后展示 diff |
| 删除大量文档 | 需人工确认 |

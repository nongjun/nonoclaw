## 示例1：AGENTS.md

```markdown
# AGENTS.md — 示例项目 AI 编码助手指南

> Cursor 原生读取此文件，是 AI 理解本项目的**唯一入口**。修改代码前请通读。

## 项目概述

本项目是一个多模块业务系统，包含 10+ 个业务模块，采用 **Monorepo** 结构。

## 技术栈

| 层 | 技术 |
|---|------|
| 后端 | Python 3.11 · FastAPI · SQLAlchemy |
| 前端 | Vue 3 · Vite 5 · Element Plus · pnpm |
| 数据库 | MySQL 8.0（utf8mb4_unicode_ci） |
| 缓存 | Redis（第三方Token共享） |
| 部署 | Docker Compose + 宿主机 Nginx |
| AI | OpenAI 兼容 API，通过 AIService 统一调用 |

## 知识库索引

> `文档/` 是唯一真相源，信息矛盾时以此为准。

- **[架构.md](架构.md)** — 子系统划分、模块依赖、容器部署拓扑、数据存储
- **[文档/设计文档/](文档/设计文档/)** — 功能设计方案（均标注验证状态）
- **[文档/核心信念/](文档/核心信念/)** — 技术栈标准、部署规范、编码约定
- **[文档/执行计划/](文档/执行计划/)** — 当前迭代任务、已完成里程碑、技术债务
- **[文档/产品规格/](文档/产品规格/)** — 各模块产品需求、业务规则
- **[文档/参考资料/](文档/参考资料/)** — API文档、第三方接口对接、数据库Schema
- **[文档/质量评分/](文档/质量评分/)** — 各模块成熟度与改动风险
- **[文档/凭据与配置/](文档/凭据与配置/)** — 第三方AppID/Secret、数据库密码、JWT密钥等
- **[文档/变更日志/](文档/变更日志/)** — 跨模块变更记录

├── 模块A/              # module-a-frontend + module-a-backend

├── 模块C/              # module-c-frontend + module-c-backend

## 核心编码规范

### 公共模块（强制使用）

| 能力 | 导入路径 | 说明 |
|------|---------|------|
| AI 调用 | `shared_backend.services.ai_service.AIService` | 禁止直接请求 AI API |
| 响应格式 | `shared_backend.middleware.response` | `success_response()`/`error_response()` |
| 前端认证 | `@shared/utils/auth` | Token 管理、Axios 拦截器 |

### 后端规范

- 使用 `async/await` 异步编程
- 密码/API Key 禁止硬编码
- 容器间通信使用内部端口

### 前端规范

- 包管理器统一 pnpm
- 字符串枚举值以后端定义为准

## 部署

- 所有服务容器化部署，前后端分离
- 仅 Nginx 层配置 SSL，后端端口绑定 127.0.0.1
- Docker 网络统一使用 app_network

## 凭据

开发环境凭据见 [文档/凭据与配置/](文档/凭据与配置/)
```

## 示例2：架构.md

```markdown
# 示例项目 — 系统架构

## 业务域划分

| 业务域 | 子系统 | 职责 | 域名 |
|--------|--------|------|------|
| 统一入口 | 门户系统 | 登录认证、模块导航、权限管理 | portal.your-domain.com |
| 客户管理 | 联系人管理 | 客户资源管理、跟进记录、AI标签 | contact.your-domain.com |
| 内容运营 | 内容中心 | 话术库、素材管理、侧边栏 | cb.your-domain.com |
| 合规审计 | 会话存档 | 第三方聊天记录拉取、查看、分析 | archive.your-domain.com |
| AI赋能 | 智能回复 | AI回复建议、上下文理解 | reply.your-domain.com |
| 营销工具 | 朋友圈管理 | 朋友圈任务、群发、营销日历 | moment.your-domain.com |
| 自动化 | 第三方托管 | RPA自动回复、技能系统、智能追问 | 规划中 |
| 裂变增长 | 老带新 | 推荐有礼、小程序 | ldx.your-domain.com |
| AI基础 | AI服务 | 统一AI调用、多模型切换 | 内部服务 |

## 代码包结构

项目根目录/
├── 模块A/             # module-a-frontend + module-a-backend
├── 模块B/             # module-b-frontend + module-b-backend
├── 模块C/             # module-c-frontend + module-c-backend
├── ...                # 其他业务模块
├── 公共模块/          # 共享组件和工具函数
└── 文档/              # 结构化知识库

## 模块依赖关系

- **门户系统** → 为所有子系统提供JWT认证，通过Redis共享第三方access_token
- **联系人/内容中心/朋友圈/智能回复** → 都依赖门户系统的认证和第三方Token
- **第三方托管** → 调用AI服务生成回复 → 依赖Redis存储系统设置和任务队列
- **会话存档** → 通过第三方SDK拉取聊天记录 → 存入MySQL → 前端展示
- **AI服务** → 统一封装OpenRouter调用，被第三方托管、智能回复、联系人(AI标签)共用
- **所有模块** → 共享同一个MySQL实例（app_mysql），通过不同表前缀隔离

## 部署拓扑

**流量路径**：用户 → DNS(*.your-domain.com) → Nginx(SSL终止) → 127.0.0.1:各端口 → Docker容器

**生产模式（Nginx静态托管）**：门户、内容中心、智能回复、朋友圈、联系人、第三方托管
**开发模式（HMR热重载）**：会话存档、撩回搭子

## 数据存储

| 存储 | 类型 | 用途 | 路径/访问 |
|------|------|------|---------|
| app_mysql | MySQL 8.0 | 所有模块业务数据 | 127.0.0.1:3307 |
| portal-redis | Redis | 第三方Token共享、托管系统设置 | 容器内部6379 |
| 系统盘 | 文件 | Docker数据、代码 | /var/lib/docker, /opt/app/ |
| 数据盘 | 文件 | 媒体上传(37G+) | /data/app/{uploads,media,assets} |
| 备份 | 文件 | 每日凌晨2点自动备份 | /data/backup/ |
```

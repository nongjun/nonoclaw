# 项目结构化文档库构建

> **本技能仅用于首次搭建**。文档库建好后的所有维护统一由文档园丁技能负责。

## 核心理念

**给 Agent 一张地图，而不是一本说明书。**

1. **渐进式披露**：`AGENTS.md` 是 Cursor 原生 AI 入口文件（≤250行），指向 `文档/` 深层文档
2. **不在仓库里 = 不存在**：把隐性知识搬入仓库
3. **为 Agent 可读性优化**：文档不含代码示例，充分信任大模型
4. **只记录长期有效的知识**：禁止创建变更日志、巡检报告、审计报告等临时性文档

## 命名规范

**一切使用中文命名**：文件、目录、导航文档、架构文档全部中文，`.md` 格式。

## 知识库目录结构

```
项目根目录/
├── AGENTS.md              # Cursor 原生 AI 入口文件（≤250行）
├── 架构.md                # 项目鸟瞰图
├── 文档/                  # 结构化知识库（唯一真相源）
│   ├── 核心信念/          # 团队工程原则与长期规范
│   ├── 设计文档/          # 仍在指导开发的架构设计
│   ├── 产品规格/          # 业务规则、判断逻辑、决策标准
│   ├── 参考资料/          # API文档、第三方对接、数据库Schema
│   ├── 凭据与配置/        # 开发阶段直接记录明文
│   ├── 操作手册/          # 操作SOP
│   └── 执行计划/
│       ├── 进行中/
│       └── 技术债务/
```

**注意**：不创建 `变更日志/`、`测试报告/`、`质量评分/`、`执行计划/已完成/` 目录。

## AGENTS.md 编写规则

Cursor 原生文件，AI 助手启动时自动读取，是 AI 理解项目的**唯一入口**。

**严格 ≤250 行**。必含：项目概述、技术栈、知识库索引、目录结构、标准模块结构、核心编码规范、部署约束、凭据位置。
**禁止放入**：详细 API 文档、完整部署步骤、数据库 Schema、代码示例。

## 架构.md 编写规则

必含：业务域划分、代码包分层、模块依赖关系、部署拓扑（容器/端口/域名）、数据存储。

## 文档分类规则

| 类型 | 说明 | 何时更新 |
|---|---|---|
| 核心信念 | 不随任务变化的长期约定 | 规则变化时 |
| 设计文档 | 仅保留仍在指导开发的架构设计，落地完成的删除 | 架构变化时 |
| 产品规格 | 业务规则、判断标准、人类审核过的决策 | 决策变化时必须同步更新 |
| 执行计划 | 进行中的任务；完成后直接删除，不归档 | 任务推进时 |
| 凭据与配置 | Token/密码/API Key 集中明文记录 | 配置变化时 |

## 初始化工作流（使用子 agent 并行）

### 阶段1：搭建结构 + 扫描现有文档（并行）

| 子 agent | 任务 |
|---------|------|
| agent-A | 创建 `文档/` 完整目录结构 |
| agent-B | 递归扫描项目下所有 `.md` 文件，生成文档清单 |

### 阶段2：编写核心文档 + 分类现有文档（并行）

| 子 agent | 任务 |
|---------|------|
| agent-C | 编写 `架构.md`（读取代码结构、docker-compose、nginx配置） |
| agent-D | 编写 `AGENTS.md`（基于架构.md和文档结构） |
| agent-E | 将系统级文档迁移到 `文档/` 对应子目录 |
| agent-F | 整合凭据信息（读取 .env，生成凭据总览） |

### 阶段3：收尾

1. 清理空目录
2. 口头汇报初始化结果（不生成报告文件）

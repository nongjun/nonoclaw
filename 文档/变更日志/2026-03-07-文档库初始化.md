## 发现的问题

1. **.cursor/ 与 瑞小美-harness/ 内容重复**：agents/、commands/、skills/ 在两处均有副本，需建立同步机制避免不一致
2. **server.ts 体量过大**：2100+ 行单文件包含全部核心逻辑，建议后续拆分
3. **测试覆盖率为零**：所有模块均无单元测试
4. **文档知识集中在 templates/**：OpenClaw 风格的身份/记忆/规则体系文档丰富，但 feishu/ 模块缺少 API 文档

## 新增的隐性知识

- templates/AGENTS.md 和项目根目录的 AGENTS.md 职责不同：前者是用户工作区的 Agent 行为指南，后者是开发者理解本项目代码的入口
- bridge.ts 是面向 OpenClaw 的兼容层，与主服务 server.ts 共享 feishu/ 和 memory 模块但独立运行
- 记忆系统分为两层：memory.ts（库）被 server.ts 实例化用于会话日志；memory-tool.ts（CLI）被 Cursor Agent 通过 shell 调用用于自主记忆

## 后续建议

1. **补充设计文档**：为 server.ts 的消息处理流程、feishu/ 模块架构、记忆系统设计编写设计文档
2. **建立 .cursor/ 与 瑞小美-harness/ 的同步机制**：可通过脚本或 git hook 保持一致
3. **补充技术债务文档**：将 server.ts 拆分、测试覆盖率等问题录入 文档/执行计划/技术债务/
4. **定期运行文档园丁**：使用文档园丁技能定期巡检文档与代码的一致性

# 驾驶舱平台产品规格

## 产品定位

**企业级 AI 助理平台**——让不懂代码的人也能用上 Cursor 级别的 AI 能力。

核心逻辑：Cursor CLI 做引擎，我们做驾驶舱。统一管理用户，通过网页 UI 提供对话式交互，本地 Agent 在员工电脑上执行实际操作。

不是"Cursor 二道贩子"，而是把 Cursor CLI 的强大能力封装成非技术人员可用的产品。

## 用户画像

### 终端用户（员工）

- 瑞小美轻医美集团高管团队
- 不懂代码，不懂 Cursor IDE
- 日常需要 AI 辅助处理工作任务
- 期望：打开网页就能对话，像用微信/ChatGPT 一样自然

### 管理员（你）

- 统一管理所有用户
- 监控用量和费用
- 配置权限和功能开关

## 核心价值

| 价值维度 | 说明 |
|---------|------|
| 降低门槛 | 非技术人员无需安装 Cursor IDE，打开网页即用 |
| 统一管理 | 权限、审计、费控——企业 IT 的刚需 |
| 定制化 | 不同岗位配不同 system prompt 和工具权限 |
| 数据本地化 | AI 操作在员工电脑本地执行，敏感数据不上传 |
| 能力强大 | 继承 Cursor CLI 的完整能力（文件操作、Shell、代码生成等） |

## 功能清单

### P0：核心功能（MVP 必须）

| 功能 | 说明 |
|------|------|
| 对话聊天 | 文字输入，流式输出，类 ChatGPT 体验 |
| Cursor CLI 调度 | 接收消息 → 调用本地 Cursor CLI → 返回结果 |
| WebSocket 通信 | 中心服务器与本地 Agent 的实时双向通信 |
| 设备在线状态 | 本机 Agent 是否在线，实时显示 |
| 基础用户认证 | 登录/注册，Agent 绑定 |

### P1：体验增强（内测阶段）

| 功能 | 说明 |
|------|------|
| 会话历史 | 对话记录持久化，可回溯 |
| 文件/图片发送 | 对话中发送附件，Agent 可接收和处理 |
| 一键安装脚本 | 员工执行一个命令完成本地 Agent 部署 |
| 断线重连 | Agent 断线自动重连，结果缓存补发 |
| 设备管理页 | 查看所有已注册设备及其状态 |

### P2：管理能力（正式阶段）

| 功能 | 说明 |
|------|------|
| 管理后台 | 用户列表、设备管理、全局配置 |
| 权限控制 | 按用户配置可用能力（文件操作、Shell 等） |
| 用量统计 | 使用次数、token 消耗、费用追踪 |
| 指令过滤 | 服务器端拦截危险指令 |
| 审计日志 | 所有操作记录可查 |

### P3：规模化（持续迭代）

| 功能 | 说明 |
|------|------|
| Windows 支持 | 本地 Agent 支持 Windows 自启动和部署 |
| 文件浏览器 | 通过 Agent 代理查看员工电脑文件 |
| 多模型支持 | 配置不同模型（Claude、GPT 等） |
| 岗位模板 | 不同岗位预配 system prompt 和工具权限 |
| 计费系统 | 按用量计费，支持部门分账 |

## 参考项目

### claudecodeui（基础代码）

- GitHub：siteboon/claudecodeui
- 许可证：GPL-3.0（商用需评估合规性）
- 已有能力：React 前端 + Express 后端 + Cursor CLI 集成 + 文件浏览器 + Shell 终端 + 会话管理
- 改造方向：从"单机工具"改造为"多用户平台"，新增中心服务器和本地 Agent 架构

### 与现有虾群平台版的关系

虾群平台版是"飞书 → Cursor CLI"的单点桥接，驾驶舱平台是其架构升级：

| 维度 | 虾群平台版 | 驾驶舱平台 |
|------|-----------|-----------|
| 入口 | 飞书 | Web UI |
| 架构 | 单机单进程 | 中心服务器 + 多本地 Agent |
| 用户 | 飞书群内成员 | 注册用户 |
| 管理 | 无 | 完整管理后台 |
| 扩展性 | 单机 | 多设备 |

## 非功能需求

| 维度 | 要求 |
|------|------|
| 安全 | 传输加密（wss://）、身份认证、指令过滤、审计日志 |
| 可用性 | Agent 断线自动重连，任务不丢失 |
| 性能 | 流式输出延迟 < 1 秒，支持 50+ 并发用户 |
| 易用性 | 一键安装本地 Agent，Web UI 零学习成本 |

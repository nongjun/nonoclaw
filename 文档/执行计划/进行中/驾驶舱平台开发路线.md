# 驾驶舱平台开发路线

## 目标

基于 claudecodeui 开源项目，构建"本地 Agent + 中心服务器 + 网页 UI"的企业级 AI 助理平台，让非技术人员通过网页对话使用 Cursor CLI 能力。

## 前置条件

- Cursor CLI 已安装且可用（`~/.local/bin/agent`）
- 阿里云服务器可部署中心服务器
- claudecodeui 源码已获取（`参考代码/claudecodeui/`）

## 第一阶段：MVP（预计 2-3 周）

验证核心链路跑通：用户在网页发消息 → 中心服务器路由 → 本地 Agent 调用 Cursor CLI → 结果回传到网页。

- [ ] **验证 Cursor CLI 授权模型** — 测试 `--api-key` 参数是否支持第三方 API Key 绕过 Cursor 订阅
- [ ] **本地 Agent 开发** — Bun + WebSocket 客户端 + Cursor CLI spawn 调用 + 流式输出
- [ ] **中心服务器开发** — Bun + WebSocket 服务器 + 简单消息路由
- [ ] **改造 claudecodeui 前端** — 剥离单机逻辑，接入中心服务器 WebSocket
- [ ] **极简对话功能** — 一个聊天框，能发消息、看流式回复
- [ ] **基础安全** — wss:// 传输、token 认证
- [ ] **仅 macOS** — 只支持你自己的机器测试

**交付物**：你能在自己电脑上通过网页和本地 Cursor CLI 对话。

## 第二阶段：内测（预计 2-3 周）

扩展到多人使用，验证多设备架构。

- [ ] **用户注册/登录** — 账号体系，JWT 认证
- [ ] **一键安装脚本** — install.sh，自动安装 Agent + 配置 + 绑定
- [ ] **设备管理** — Agent 注册、心跳上报、在线状态
- [ ] **会话历史** — 对话记录持久化到中心服务器
- [ ] **断线重连** — 指数退避重连 + 结果本地缓存 + 重连补发
- [ ] **完善 Web UI** — 历史记录页、设备状态页
- [ ] **内测** — 给 3-5 个员工部署测试

**交付物**：3-5 个员工可以通过网页使用，各自电脑上的 Agent 独立工作。

## 第三阶段：正式上线（持续迭代）

完善管理能力和扩展性。

- [ ] **管理后台** — 用户列表、设备管理、全局配置
- [ ] **权限控制** — 按用户配置能力范围
- [ ] **用量统计** — 使用次数、token 消耗趋势
- [ ] **指令过滤** — 服务器端拦截危险指令
- [ ] **审计日志** — 所有操作记录可查
- [ ] **Windows 支持** — NSSM 自启动、install.exe 安装包
- [ ] **文件浏览器** — 通过 Agent 代理查看远程文件
- [ ] **岗位模板** — 不同岗位预配 system prompt
- [ ] **计费系统** — 按用量计费，部门分账

**交付物**：可对外提供的企业级产品。

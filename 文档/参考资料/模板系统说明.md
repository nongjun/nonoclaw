# 模板系统说明

`templates/` 目录包含龙虾军团的完整模板体系，用于初始化新项目或新用户工作区。以下是各模板文件的用途和相互关系。

## 顶层文件

| 文件 | 用途 |
|------|------|
| **AGENTS.md** | 用户工作区的 Agent 指南，Cursor 自动加载此文件作为上下文 |

## .cursor/ 目录下的核心模板

### 身份与人格

| 文件 | 用途 |
|------|------|
| **SOUL.md** | AI 的人格定义/灵魂设定，决定 AI 的沟通风格与价值观 |
| **IDENTITY.md** | 身份元数据，包含 AI 的名称、版本、角色等基本信息 |
| **USER.md** | 主人信息，记录 AI 服务对象的偏好、背景和需求 |

### 记忆与启动

| 文件 | 用途 |
|------|------|
| **MEMORY.md** | 长期记忆骨架，AI 跨会话保留的关键信息结构 |
| **BOOT.md** | 启动自检清单，每次会话开始时 AI 执行的检查项 |
| **BOOTSTRAP.md** | 首次启动仪式，新用户/新项目初始化时的引导流程 |
| **HEARTBEAT.md** | 心跳检查清单，定期执行的健康检查和维护任务 |

### 任务与工具

| 文件 | 用途 |
|------|------|
| **TASKS.md** | 定时任务文档，记录需要周期性执行的自动化任务 |
| **TOOLS.md** | 工具能力清单，列出 AI 可调用的所有工具及其使用场景 |

### 规则文件（.cursor/rules/*.mdc）

共 9 个规则文件，定义编码规范和安全标准：

| 文件 | 作用域 |
|------|--------|
| **基础规则.mdc** | 全局基础约束，始终生效 |
| **安全红线.mdc** | 安全底线，始终生效 |
| **技术栈标准.mdc** | 技术选型与部署规范 |
| **通用编码风格.mdc** | 跨语言通用编码风格 |
| **Go编码风格.mdc** | Go 语言编码风格 |
| **Go安全规范.mdc** | Go 语言安全规范 |
| **Python编码风格.mdc** | Python 编码风格 |
| **Python安全规范.mdc** | Python 安全规范 |
| **Swift编码风格.mdc** | Swift 编码风格 |

### 技能模板（.cursor/skills/）

技能目录包含可复用的专业能力模板，每个技能是一个独立目录，包含 `SKILL.md` 描述文件和可选的脚本、参考资料。技能在特定场景下被自动激活。

## 文件间的关系

```
AGENTS.md（入口）
  ├── 加载 IDENTITY.md → 确定"我是谁"
  ├── 加载 SOUL.md → 确定"我的性格"
  ├── 加载 USER.md → 确定"我服务谁"
  ├── 加载 MEMORY.md → 恢复长期记忆
  ├── 执行 BOOT.md → 自检启动
  │     └── 首次？→ 执行 BOOTSTRAP.md
  ├── 加载 rules/*.mdc → 应用编码规范
  ├── 加载 skills/ → 注册可用技能
  ├── 加载 TOOLS.md → 注册工具能力
  └── 周期执行 HEARTBEAT.md / TASKS.md → 持续维护
```

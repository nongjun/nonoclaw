# nonoclaw 安装帮助 — 服务器版（中国大陆）

本文档记录在中国大陆 Linux 服务器上部署 nonoclaw + Cursor CLI 的完整过程，重点涵盖 VPN 翻墙方案的搭建和精确控制。

- Linux 服务器（本文以阿里云 Alibaba Cloud Linux 3 / x86_64 为例）
- 一个可用的代理订阅地址（Clash 兼容格式）
- Cursor 账号及 API Key

## 1. 核心问题

Cursor CLI 在中国大陆服务器上遇到两个障碍：

1. **网络不通** — Cursor 的 API 服务器（cursor.sh、anthropic.com 等）被 GFW 封锁
2. **区域限制** — 即使通过 HTTP 代理翻墙，Claude/GPT/Gemini 模型仍报 "Model not available: This model provider doesn't serve your region"

根因是 Cursor 的区域检测依赖 **DNS 解析**，纯 `HTTP_PROXY` 只代理了 HTTP 连接，DNS 仍走本地解析（返回中国 IP）。解决方案是 `proxychains` 的 `proxy_dns` 选项，让 DNS 查询也走代理节点。

## 2. Mihomo 代理服务

### 安装 Mihomo

```bash
mkdir -p /opt/clash
# GitHub 直连慢，用镜像
curl -sL "https://gh-proxy.com/https://github.com/MetaCubeX/mihomo/releases/download/v1.19.8/mihomo-linux-amd64-v1.19.8.gz" -o /opt/clash/mihomo.gz
gunzip -f /opt/clash/mihomo.gz
chmod +x /opt/clash/mihomo
```

### 下载 GeoIP 数据

```bash
cd /opt/clash
curl -sL "https://gh-proxy.com/https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/country.mmdb" -o country.mmdb
curl -sL "https://gh-proxy.com/https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geosite.dat" -o GeoSite.dat
curl -sL "https://gh-proxy.com/https://github.com/MetaCubeX/meta-rules-dat/releases/download/latest/geoip.dat" -o GeoIP.dat
```

### 拉取订阅

```bash
echo -n "你的订阅URL" > /opt/clash/.subscription_url
chmod 600 /opt/clash/.subscription_url

curl -sL -A "clash-verge/v2.2.0" "$(cat /opt/clash/.subscription_url)" -o /opt/clash/subscription_raw.yaml
chmod 600 /opt/clash/subscription_raw.yaml
```

### 生成精确规则配置

将配置生成脚本放在 `/opt/clash/.gen_config.py`，核心策略：

- **节点过滤**：只保留日本、台湾、美国节点，排除香港和大陆
- **自动测速**：`url-test` 类型自动选最快节点，优先级为日本 > 台湾 > 美国
- **精确规则**：只有 Cursor 相关域名走代理，其余全部 DIRECT

域名白名单：
```
*.cursor.sh / *.cursor.com / *.cursorapi.com / *.cursor-cdn.com
*.anthropic.com / *.openai.com / *.googleapis.com / *.google.com
*.sentry.io
任何包含 cursor 关键词的域名
兜底规则：MATCH → DIRECT
```

运行生成：
```bash
python3 /opt/clash/.gen_config.py
chmod 600 /opt/clash/config.yaml
```

### systemd 服务

创建 `/etc/systemd/system/mihomo.service`：

```ini
[Unit]
Description=Mihomo Proxy (Cursor CLI only)
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/opt/clash/mihomo -d /opt/clash
Restart=on-failure
RestartSec=5
LimitNOFILE=65535
ProtectSystem=strict
ReadWritePaths=/opt/clash
ProtectHome=true
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable --now mihomo
```

### 验证

```bash
# 代理端口应仅监听 127.0.0.1
ss -tlnp | grep 7897
# 测试代理连通
curl -x http://127.0.0.1:7897 https://www.google.com -o /dev/null -w "%{http_code}"
```

### 订阅更新

```bash
/opt/clash/update-subscription.sh
```

## 3. Cursor CLI

### 安装

```bash
HTTPS_PROXY=http://127.0.0.1:7897 HTTP_PROXY=http://127.0.0.1:7897 \
  curl https://cursor.com/install -fsS | bash
```

安装后 `agent` 命令位于 `~/.local/bin/agent`。确保 PATH 包含该目录：

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 创建代理 wrapper

直接调用 `agent` 会因区域限制失败。需要用 wrapper 脚本让所有流量（含 DNS）走代理。

重命名原始二进制：
```bash
mv ~/.local/bin/agent ~/.local/bin/agent-real
```

创建 wrapper `~/.local/bin/agent`：
```bash
#!/bin/bash
export HTTPS_PROXY=http://127.0.0.1:7897
export HTTP_PROXY=http://127.0.0.1:7897
exec proxychains4 -q -f /opt/clash/proxychains.conf "$(dirname "$0")/agent-real" "$@"
```

```bash
chmod +x ~/.local/bin/agent
```

## 4. proxychains 解决区域限制

这是本方案中**最关键的一步**。

### 问题

纯 `HTTP_PROXY` 环境变量只能解决网络连通问题（`auto` 模型可用），但指定特定模型（如 `claude-4.6-opus-high-thinking`）时仍报区域限制。原因：

- `HTTP_PROXY` 只代理 HTTP/HTTPS 连接
- DNS 解析仍走本地，Cursor 后端通过 DNS 结果判断区域
- 等效于你在电脑上用「系统代理」模式而非「TUN 模式」

### 解决

安装 proxychains-ng（LD_PRELOAD 级别的 socket 劫持，等效于进程级 TUN）：

```bash
yum install -y proxychains-ng  # CentOS/RHEL/Alibaba Cloud Linux
# 或 apt install -y proxychains4  # Debian/Ubuntu
```

创建 `/opt/clash/proxychains.conf`：

```
strict_chain
proxy_dns
remote_dns_subnet 224
tcp_read_time_out 15000
tcp_connect_time_out 8000

[ProxyList]
socks5 127.0.0.1 7897
```

关键配置项：**`proxy_dns`** — 让 DNS 查询也走 SOCKS5 代理，这样 Cursor 后端看到的 DNS 解析结果来自代理节点（日本/台湾/美国），而非中国。

### 验证

```bash
export CURSOR_API_KEY="你的key"
agent -p --trust --model claude-4.6-opus-high-thinking "回复两个字：成功"
# 应该返回 "成功"
```

## 5. 精确规则控制

### 三层隔离架构

| 层级 | 机制 | 效果 |
|------|------|------|
| 进程层 | `agent` wrapper + proxychains | 只有 `agent` 命令的流量进入代理 |
| 端口层 | Mihomo 仅监听 127.0.0.1:7897 | 外部设备无法连入 |
| 规则层 | 精确域名白名单 + `MATCH,DIRECT` | 即使流量到了代理，也只有 Cursor 相关域名走 VPN |

这确保了：
- 服务器上的其他服务（Docker、Nginx、SSH）完全不受影响
- 没有全局 `HTTP_PROXY`、没有 TUN 设备、没有路由表修改
- Mac 终端 SSH 连入后 `env | grep proxy` 为空

## 6. nonoclaw

### 安装 Bun

```bash
# GitHub 直连慢，用镜像
curl -sL "https://gh-proxy.com/https://github.com/oven-sh/bun/releases/latest/download/bun-linux-x64.zip" -o /tmp/bun.zip
cd /tmp && unzip -o bun.zip
mv bun-linux-x64/bun /usr/local/bin/bun
chmod +x /usr/local/bin/bun
rm -rf /tmp/bun.zip /tmp/bun-linux-x64
```

### 拉取代码

```bash
cd /root
HTTPS_PROXY=http://127.0.0.1:7897 HTTP_PROXY=http://127.0.0.1:7897 \
  git clone https://github.com/nongjun/feishu-cursor-claw.git nonoclaw
cd nonoclaw
HTTPS_PROXY=http://127.0.0.1:7897 HTTP_PROXY=http://127.0.0.1:7897 \
  bun install
```

### 关键适配：工作区隔离

原项目 `server.ts` 中 `ROOT = resolve(import.meta.dirname, "..")`，会将工作区指向上级目录（`/root/`），导致 nonoclaw 的 `.cursor/rules`、`AGENTS.md` 等与服务器现有配置冲突。

修改为：
```typescript
const ROOT = resolve(import.meta.dirname);  // 工作区 = /root/nonoclaw/ 自身
```

同样修改 `bridge.ts` 中的 workspace 路径。

### 配置 .env

```bash
cp .env.example .env
chmod 600 .env
# 编辑 .env 填入 CURSOR_API_KEY、FEISHU_APP_ID、FEISHU_APP_SECRET
```

### 配置项目路由

创建 `/root/nonoclaw/projects.json`：

```json
{
  "projects": {
    "server": { "path": "/root/nonoclaw", "description": "服务器管理（默认）" },
    "aios": { "path": "/root/瑞小美AiOS", "description": "瑞小美 AiOS" }
  },
  "default_project": "server"
}
```

### systemd 服务

创建 `/etc/systemd/system/nonoclaw.service`：

```ini
[Unit]
Description=Nonoclaw - Feishu remote control for Cursor CLI
After=network-online.target mihomo.service
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/root/nonoclaw
ExecStart=/usr/local/bin/bun run start.ts
Restart=on-failure
RestartSec=10
Environment=HOME=/root
Environment=PATH=/root/.local/bin:/usr/local/bin:/usr/bin:/bin

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable nonoclaw
# 飞书凭据配置好后再启动：
# systemctl start nonoclaw
```

## 8. 踩坑记录

### GitHub 下载超时

中国大陆服务器直连 GitHub 极慢或超时。解决：用 `gh-proxy.com` 镜像前缀。

### 纯 HTTP_PROXY 无法解决区域限制

`HTTP_PROXY` 只代理 HTTP 连接，DNS 仍走本地。Cursor 通过 DNS 结果判断区域。必须用 `proxychains` 的 `proxy_dns` 让 DNS 也走代理。

### Cursor CLI 安装器需要 trust 参数

首次运行 `agent -p` 会提示 "Workspace Trust Required"，需要加 `--trust` 参数。

### Bun 安装脚本依赖 GitHub

`curl https://bun.sh/install | bash` 内部会访问 GitHub releases，需要走代理或用镜像手动下载 zip。

### nonoclaw 工作区冲突

原项目 `ROOT = resolve(import.meta.dirname, "..")` 会将工作区指向上级目录，导致 nonoclaw 的 `.cursor/rules` 与宿主机现有规则冲突。修改为 `resolve(import.meta.dirname)` 实现隔离。

### projects.json 位置

原项目读取 `resolve(ROOT, "projects.json")`。修改 ROOT 后，`projects.json` 应放在 nonoclaw 目录内而非上级目录。
